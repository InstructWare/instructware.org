## Contrast

- contrast.section_title: Traditional Software vs InstructWare
- contrast.legacy.title: Traditional Software (No IWP)
- contrast.legacy.point_1: Intent is scattered and source of truth is unstable.
- contrast.legacy.point_2: Delivery is possible, but alignment relies heavily on manual review and repeated cross-team sync.
- contrast.legacy.point_3: Runtime evolution is mostly release-driven and slower.
- contrast.iw.title: InstructWare Runtime (With IWP)
- contrast.iw.point_1: Shared intent artifacts provide a stable source of truth.
- contrast.iw.point_2: Governed validation and traceability keep implementation aligned with intent.
- contrast.iw.point_3: Continuous runtime evolution reduces coordination overhead at scale.

## Laws

- laws.section_title: The Three Laws of InstructWare
- laws.item_1: Natural language is the primary source of intent; the Intent Layer is language-agnostic.
- laws.item_2: Interfaces are dynamic, adaptive, and governable.
- laws.item_3: Runtime is continuously evolvable; the Execution Layer is implementation-language-agnostic.

## Values

- values.section_title: Values
- values.item_1: Human instruction as the contract over code as the bottleneck.
- values.item_2: Instruction-to-execution pipelines over release-bound feature queues.
- values.item_3: Source-traceable runtime evolution over opaque one-shot deployments.
- values.item_4: Governed machine adaptation to human goals over humans adapting to machine constraints.
