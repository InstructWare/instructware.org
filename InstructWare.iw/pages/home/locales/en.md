## Contrast

- contrast.section_title: Static Software -> Dynamic Software Systems
- contrast.legacy.title: Static Software
- contrast.legacy.point_1: Human intent usually reaches the system only after passing through specs, tickets, and code handoffs.
- contrast.legacy.point_2: Software changes mainly through release cadence instead of evolving continuously within governance boundaries.
- contrast.legacy.point_3: Even when AI accelerates code generation, systems still lose coherence once intent and implementation drift apart.
- contrast.iw.title: InstructWare Runtime
- contrast.iw.point_1: Natural language becomes a durable bridge between humans and software behavior.
- contrast.iw.point_2: IWP keeps the relationship between intent, code, and runtime behavior clear and continuously inspectable.
- contrast.iw.point_3: Software can then evolve with a better chance of remaining reviewable, governable, and maintainable.

## Laws

- laws.section_title: The Three Laws of InstructWare
- laws.item_1: Natural language is the primary source of intent, and the Intent Layer remains language-agnostic.
- laws.item_2: Interfaces can be dynamic and adaptive while remaining governable.
- laws.item_3: Runtime remains continuously evolvable, and the Execution Layer stays implementation-language-agnostic.

## Values

- values.section_title: Values
- values.item_1: Human instruction as the contract, rather than code becoming the bottleneck.
- values.item_2: Instruction-to-execution pipelines, rather than feature queues bound to release cycles.
- values.item_3: Source-traceable runtime evolution over opaque one-shot deployments.
- values.item_4: Governed machine adaptation to human goals, rather than humans adapting to machine constraints.
